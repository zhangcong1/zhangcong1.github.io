# 3.2 Git Hooks

  • pre-commit
    git commit 执行前
    它不接受任何参数，并且在获取提交日志消息并进行提交之前被调用。
    脚本 git comit 以非零状态退出会导致命令在创建提交之前中止。
  • commit-msg
    git commit 执行前
    可用于将消息规范化为某种项目标准格式。
    还可用于在检查消息文件后拒绝提交。
    简单来说这两个钩子：
  1. commit-msg：可以用来规范化标准格式，并且可以按需指定是否要拒绝本次提交
  2. pre-commit：会在提交前被调用，并且可以按需指定是否要拒绝本次提交

说明：可以用 git commit --no-verify 绕过
