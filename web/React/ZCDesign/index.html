# TypeScript

# 一、概念：编程语言类型

  • 动态类型语言

    在运行期间才会去做数据类型检查。不用指定变量类型。

  • 静态类型语言

    在编译时进行数据类型检查。

# 为什么使用 TypeScript？

  1. 程序更容易理解

    问题：函数或者方法输入输出的参数类型，外部条件等。

    动态语言的约束：需要手动调试等过程。

  2. 效率更高

    在不同代码块和定义中进行跳转。

    代码自动补全

    丰富的接口提示

  3. 更少的错误

    编译期间能够发现大部分错误

    杜绝一些比较常见的错误 例如 传入 0，布尔值判断为 false

  4. 非常好的包容性

    完全兼容 JavaScript

    第三方库可以单独编写类型文件

    流行项目都支持 TypeScript - React，Vue，Ant Design
